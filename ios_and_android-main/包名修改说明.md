## 下一步操作

### 必须清理并重新构建！

由于包名已更改，你必须清理旧的构建文件并重新构建应用：

```bash
# 方法 1: 使用清理脚本
bash clean-and-rebuild.sh

# 方法 2: 手动清理
cd /Users/yankun/my_project/ios_app_demo/store

# 清理 Android 构建
rm -rf android/app/build
rm -rf android/build
rm -rf android/.gradle

# 清理 Gradle 缓存
rm -rf ~/.gradle/caches/transforms-*

# 停止 Gradle
cd android
./gradlew --stop
./gradlew clean
cd ..

# 清理 Expo 缓存
rm -rf .expo .metro

# 重新构建
npx expo run:android
```

### 卸载旧应用（重要！）

如果设备上已安装旧包名的应用，需要先卸载：

```bash
# 卸载旧包名的应用
adb uninstall com.yankun.store

# 或者在设备上手动卸载

# 然后安装新包名的应用
npx expo run:android
```

### iOS 构建（如果需要）

如果要在 iOS 上构建，也需要清理：

```bash
cd ios
pod deintegrate
pod install
cd ..

npx expo run:ios
```

## 验证修改

构建完成后，可以通过以下方式验证：

```bash
# 检查新包名应用是否已安装
adb shell pm list packages | grep com.wfkj.store

# 查看应用信息
adb shell dumpsys package com.wfkj.store | grep versionName
```

## 注意事项

1. ⚠️ **不同的包名会被视为不同的应用**
   - 无法直接升级，需要卸载旧应用
   - 应用数据会丢失

2. ⚠️ **如果已发布到应用商店**
   - 修改包名后无法作为更新发布
   - 会被视为新应用

3. ✅ **开发阶段修改包名是安全的**
   - 适合在正式发布前修改
   - 建议清理所有缓存重新构建
